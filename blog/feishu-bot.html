飞书 Bot 开发实战：从零搭建一个智能工作流

飞书

飞书开放平台的生态越来越完善，通过 Bot 能力，我们可以将 AI 无缝融入日常工作流。本文记录了我从零搭建一个飞书智能 Bot 的完整过程。

项目背景

团队日常有大量重复性的信息处理工作：整理会议纪要、汇总周报、回答常见问题等。我想做一个飞书 Bot，能够自动处理这些任务，让团队成员把时间花在更有价值的事情上。

第一步：创建飞书应用

在飞书开放平台（open.feishu.cn）创建一个企业自建应用：

  • 填写应用名称和描述
  • 选择「机器人」能力
  • 配置权限：消息接收、消息发送、文档读取等
  • 获取 App ID 和 App Secret

第二步：搭建后端服务

我使用 Python + FastAPI 作为后端框架，结构简洁且开发效率高。

第三步：实现事件订阅

飞书通过 Webhook 推送事件到我们的服务。核心流程：

  • 接收飞书 POST 请求
  • 验证签名（确保请求来自飞书）
  • 解密消息内容
  • 调用 LLM 处理
  • 通过飞书 API 回复消息

第四步：集成 LLM

这是 Bot 的「大脑」。根据不同的消息类型路由到不同的处理逻辑：

  • 普通消息：直接调用 GPT-4o 生成回复
  • 文档分享：读取文档内容，生成摘要
  • 日历事件：自动生成会议纪要模板

踩坑记录

  • 飞书的事件订阅需要在 3 秒内返回 200，否则会重试。长时间任务需要异步处理
  • 权限申请流程较严格，建议提前规划好所需权限一次性申请
  • 消息卡片的 JSON Schema 验证比较严格，建议先在飞书卡片搭建器中调试
  • Token 有过期时间，需要做好自动刷新机制

总结

飞书 Bot 开发的门槛比想象中低很多，官方文档也比较完善。关键是要想清楚你的 Bot 要解决什么问题，然后从最小可行产品开始迭代。不要一开始就追求大而全，先做一个能跑起来的版本，再逐步增加功能。
